# نظام الدفع وإدارة الديون

## نظرة عامة
تم تطوير نظام ذكي لإدارة المدفوعات والديون في صفحة تسجيل الخدمات، يتكامل مع سجل الحجوزات والديون السابقة للعميل.

## الميزات الرئيسية

### 1. البحث التلقائي عن العميل
- عند إدخال رقم هاتف مكون من 10 أرقام، يتم البحث التلقائي عن العميل
- إذا تم العثور على العميل، يتم:
  - ملء اسم العميل تلقائياً
  - عرض الديون السابقة
  - عرض الحجوزات النشطة
  - عرض ملاحظات خاصة بالعميل

### 2. عرض الديون السابقة
عند العثور على العميل مع ديون سابقة، يتم عرض:
- إجمالي الديون
- تفاصيل كل دين (التاريخ، المبلغ)
- حالة الدين
- تاريخ الاستحقاق

### 3. عرض الحجوزات النشطة
- عرض جميع الحجوزات المؤكدة أو قيد الانتظار
- تاريخ ووقت الحجز
- حالة الحجز

### 4. معالجة المدفوعات الذكية

#### السيناريو 1: دفع كامل للخدمة الحالية فقط
```
سعر الخدمة: 1000 دج
المبلغ المدفوع: 1000 دج
النتيجة: ✓ دفع كامل - لا توجد ديون
```

#### السيناريو 2: دفع جزئي (دين جديد)
```
سعر الخدمة: 1000 دج
المبلغ المدفوع: 600 دج
الدين الجديد: 400 دج
```

#### السيناريو 3: دفع زائد مع وجود ديون سابقة
```
سعر الخدمة: 1000 دج
الديون السابقة: 500 دج
المبلغ المدفوع: 1300 دج

الحسابات:
1. فائض الدفع = 1300 - 1000 = 300 دج
2. يتم خصم الفائض من الديون القديمة
3. الدين الجديد = 500 - 300 = 200 دج

النتيجة:
✓ تم سداد 300 دج من الديون القديمة
⚠️ الدين المتبقي: 200 دج
```

#### السيناريو 4: دفع زائد يغطي كل الديون
```
سعر الخدمة: 1000 دج
الديون السابقة: 500 دج
المبلغ المدفوع: 1600 دج

الحسابات:
1. فائض الدفع = 1600 - 1000 = 600 دج
2. سداد كامل الديون القديمة = 500 دج
3. فائض إضافي = 600 - 500 = 100 دج

النتيجة:
✓ تم سداد كامل الديون
💰 فائض الدفع: 100 دج
```

## قاعدة البيانات

### جداول متعلقة

#### Client
```prisma
model Client {
  client_id String  @id @default(uuid())
  name      String
  phone     String?
  notes     String?
  
  services     ServiceAction[]
  debts        Debt[]
  reservations Reservation[]
}
```

#### Debt
```prisma
model Debt {
  debt_id   String    @id @default(uuid())
  client_id String
  date_reg  DateTime
  date_exp  DateTime?
  status    String    // pending, paid, cancelled
  debt_val  Float
  
  client Client @relation(fields: [client_id], references: [client_id])
}
```

#### Reservation
```prisma
model Reservation {
  reservation_id String   @id @default(uuid())
  client_id      String
  salon_id       String
  date           DateTime
  status         String   // pending, confirmed, completed, cancelled
  
  client Client @relation(fields: [client_id], references: [client_id])
  salon  Salon  @relation(fields: [salon_id], references: [salon_id])
}
```

## كيفية الاستخدام

### 1. إضافة خدمة لعميل جديد
1. أدخل رقم الهاتف (10 أرقام)
2. أدخل اسم العميل يدوياً
3. أضف الخدمات والموظفين
4. أدخل المبلغ المدفوع
5. النظام سيحسب الدين تلقائياً

### 2. إضافة خدمة لعميل مسجل
1. أدخل رقم الهاتف (10 أرقام)
2. **سيتم ملء الاسم تلقائياً**
3. **ستظهر معلومات الديون والحجوزات**
4. أضف الخدمات
5. أدخل المبلغ المدفوع
6. النظام سيحسب:
   - الدين الجديد للخدمة الحالية
   - تطبيق الفائض على الديون القديمة
   - الدين الإجمالي الجديد

### 3. سداد ديون قديمة عند تقديم خدمة جديدة
1. أدخل رقم هاتف العميل الذي لديه ديون
2. ستظهر الديون السابقة بوضوح
3. عند إدخال مبلغ أكثر من سعر الخدمة:
   - يتم خصم سعر الخدمة أولاً
   - الفائض يذهب لسداد الديون القديمة
   - يتم عرض تفاصيل السداد في رسالة التأكيد

## التحسينات المستقبلية

### المقترحات
1. **المقدم من الحجوزات**:
   - إضافة حقل `advance_payment` لجدول Reservation
   - خصم المقدم تلقائياً عند تسجيل الخدمة

2. **تاريخ الديون**:
   - تتبع تاريخ كل عملية سداد
   - عرض سجل كامل للمدفوعات

3. **إشعارات**:
   - إشعار تلقائي عند اقتراب موعد استحقاق الدين
   - تذكير بالديون المتأخرة

4. **تقارير**:
   - تقرير شامل بالعملاء الذين لديهم ديون
   - تقرير بالديون المستحقة
   - تقرير بالديون المسددة

## أمثلة عملية

### مثال 1: عميل جديد - دفع جزئي
```
رقم الهاتف: 0551234567
اسم العميل: أحمد محمد
الخدمة: قص شعر (500 دج)
المبلغ المدفوع: 300 دج

النتيجة:
✓ تم تسجيل الخدمة
⚠️ دين جديد: 200 دج
```

### مثال 2: عميل قديم - لديه دين سابق
```
رقم الهاتف: 0551234567 (تم العثور عليه)
اسم العميل: أحمد محمد (تلقائي)
الدين السابق: 200 دج
الخدمة الجديدة: صبغة (800 دج)
المبلغ المدفوع: 1000 دج

الحسابات:
1. سعر الخدمة: 800 دج
2. فائض الدفع: 1000 - 800 = 200 دج
3. سداد الدين القديم: 200 دج

النتيجة:
✓ تم تسجيل الخدمة
✓ تم سداد كامل الديون
💰 لا توجد ديون متبقية
```

### مثال 3: عميل قديم - دفع يغطي جزء من الدين
```
رقم الهاتف: 0559876543
الدين السابق: 500 دج
الخدمة الجديدة: 1000 دج
المبلغ المدفوع: 1200 دج

الحسابات:
1. سعر الخدمة: 1000 دج
2. فائض: 200 دج
3. سداد من الدين القديم: 200 دج
4. دين متبقي: 500 - 200 = 300 دج

النتيجة:
✓ تم تسجيل الخدمة
✓ تم سداد 200 دج من الديون القديمة
⚠️ الدين المتبقي: 300 دج
```

## الملاحظات الفنية

### الأمان
- التحقق من صحة رقم الهاتف
- التحقق من عدم تجاوز المبلغ المدفوع بدون ديون سابقة
- معالجة الأخطاء بشكل صحيح

### الأداء
- البحث التلقائي فقط عند 10 أرقام
- تحميل البيانات بشكل تدريجي
- عرض مؤشر التحميل أثناء البحث

### تجربة المستخدم
- ملء تلقائي للاسم عند العثور على العميل
- عرض واضح للديون والحجوزات
- رسائل تفصيلية عند التسجيل
- ألوان تمييزية (أحمر للديون، أخضر للسداد)
